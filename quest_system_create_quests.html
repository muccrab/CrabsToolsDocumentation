Crabs Quest System - Creating Your Own Quest

Crabs Quest System uses a JSON-based structure to define quests. Each quest is composed of parts, and each part contains tasks. Below is a breakdown of how to define your own quest.

Basic Structure

Here's a simple example of a quest definition in JSON: 

{
    "name": "Quest 1",
    "quest_parts": [{
        "name": "Introduction",
        "tasks": [{
            "name": "Meet the Mentor",
            "type": "basicTask",
            "data": { "points": 10 }
        },{
            "name": "Learn the Basics",
            "type": "basicTask",
            "data": { "points": 20 }
        }]
    }, {
        "name": "Gathering Supplies",
        "status": "inactive",
        "tasks": [{
            "name": "Collect Wood",
            "type": "basicTask",
            "data": { "points": 15 }
        },{
            "name": "Find Food",
            "type": "basicTask",
            "data": { "points": 25 }
        }]
    }]
}

Quest Properties

  • name: The title of the quest, used for identifying and displaying the quest.
  • quest_parts: An array of objects, each representing a segment of the quest.

Quest Part Properties

  • name: The name of the quest part.
  • status (optional): Determines the initial state of the part. Possible values are active, inactive, or omitted (defaults to active).
  • tasks: An array of tasks that must be completed in this part.

Task Properties

  • name: The label shown to the player for this task.
  • type: Type of task. Must match a registered task type in the quest system (e.g., basicTask).
  • data: Task-specific configuration. For example, points might indicate how much XP this task gives or its weight.

How to Register Custom Tasks

If you create a custom task class in C++, register it using the QuestFactory so the system can instantiate it based on the type field in JSON. Each task class must inherit from the base Task class and implement required logic.

Tips

  • You can chain parts together using custom logic in QuestPart conditions (see advanced setup).
