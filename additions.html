Crabs Shared Utility Library

To provide shared functionality across Crabs systems, a header-only library was created. Initially meant for small tools, it had a surprisingly large impact on development.

Utility Functions

  • Vector Utilities: Mapping between two vectors, splitting vectors, extracting keys from maps.
  • String Utilities: Splitting strings based on custom logic, etc.
  • Time Utilities: Provides UTC-based time formatting.
  • Switcher: A helper for flipping between two objects—good for toggle logic like turn-based systems or card battles.

Best Additions

Indexer

Indexer acts as a memory container that tracks deleted entries to avoid unnecessary memory waste. Think of it like a twisted library system:

  • Each object added gets a unique ID and is considered “owned” by the library.
  • Handles can be distributed to represent temporary access—but the library maintains ownership.
  • If no one is responsible for the handle, the item gets deleted.
  • Supports iteration by the owning system (like a librarian flipping through the entire catalog).

MultiIndexer

An extended version of Indexer that supports grouping. Each object can belong to a specific group, and systems can iterate over individual groups selectively.

Signal

Built as a wrapper over Indexer, this class stores instances of std::function<void(Args...)>, Can be used to register your own signal endpoints and can emit the signal. If the handle is lost, the enpoint is as well, so there won't be any phantom signals.

TaskQueue

A task queue that processes tasks sequentially on a dedicated thread. It uses condition variables to maintain non-blocking behavior while staying mostly sequential in logic—ideal for offloading minor tasks without halting the game loop.
