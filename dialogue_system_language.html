Crabs Dialogue System – Language

A Dialogue System language file looks like this:

As you can see, every line starts with a keyword. This keyword defines what happens on that line. The most special keyword is branch, which defines a section of code that runs in its own scope with local variables. A branch ends when another branch begins or at the end of the file.

Line indentation is only for the user’s convenience to visually distinguish branches. It has no impact on how the code is executed (no Python nonsense here).

Every file must include a root branch, which is the entry point. Even files with only one branch must declare a root.

Each line can only represent one command. Empty lines are ignored. Everything after the # symbol is treated as a comment and ignored.

Arguments are separated by spaces, unless they are enclosed in quotes.

The following commands are currently supported:

Command - climb

Jumps to the specified branch with optional arguments.

Command - climb_random

Randomly selects one of the provided branches to climb into.

Command - climb_switch

Selects a branch to climb into based on the value of the first argument.

Command - listen

Waits for user input and halts execution until a response is given. It provides users with the options if provided

Command - listen_brother

Same as listen, but also sets the speaking character.

Command - collect

Evaluates a value and stores it in a named variable.

Command - debug

Logs the evaluated arguments to the output stream.

Command - say

Outputs dialogue and pauses execution.

Command - say_brother

Like say, but includes the speaker's name.

Command - trigger

Calls one or more triggers by name.

Command - return

Ends the current branch and resumes after it.

Command - exit

Immediately terminates the dialogue by clearing the stack.
