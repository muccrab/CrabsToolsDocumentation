Crabs Save System (Godot Wrapper)

The Crabs Save System is accessible in Godot via a simple wrapper class. All functionality is available through the SaveSystemCPP wrapper.

Getting Started

  • Inherit from SaveSystemCPP.
  • In your _ready() function, call setupCPP().

Querying Saves

Use getSaveInfoCPP() to fetch a list of all save entries:

[
    {
        "name": "Save 1",
        "id": 0,
        "date": "2024-06-30T15:00:00Z"
    },
    ...
]

You can also check if there are any saves using isEmptyCPP(), which returns a boolean.

Creating and Loading Saves

  • To start a new game, call newGameCPP("SaveName"). (Note: This does not create a save file until you call saveCPP().)
  • To load a save, call loadSave(id) using the save's ID from getSaveInfoCPP().

Working with Save Data

  • saveToClass(class_name, id, object) – Save a specific object into a class (class in this context means subfolder in save location).
  • loadFromClass(class_name, id) – Load an object from the specified class.

These methods only update internal cache. To commit changes to disk, call saveCPP().

All saves are cached internally. Manual saving through saveCPP() is required to persist the changes to disk.
