Detailed Description

Provides a generic indexer class for managing objects with handles and snapshot-based iterators.

This header defines the crabs::Indexer template class, which manages a collection of objects of type TType using handles and supports efficient allocation and deallocation. The indexer provides snapshot-based iterators, meaning that iterators do not reflect changes to the newly created objects after their creation.

Main components:

  • Indexer<TType>: Manages objects, supports creation and freeing of objects, and provides iterator access.
  • IndexHandle<TType>: Represents a handle to an object managed by the indexer. Handles are move-only and automatically release their resources when destroyed.
  • IndexerIterator<TType, TConst>: Forward iterator for traversing objects in the indexer. Iterators are snapshot-based and should not be stored for long-term use.

Usage notes:

  • Iterators are snapshot-based; always create new iterators for each iteration.
  • Iterators do not register newly added objects, but they still can't access deleted objects
  • Handles are move-only and automatically release their resources when destroyed.
  • The indexer uses versioning to ensure handles and iterators remain valid only for the lifetime of the objects.
  • Thread safety is not guaranteed.
