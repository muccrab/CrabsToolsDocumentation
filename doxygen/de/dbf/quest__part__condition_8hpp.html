Detailed Description

Contains definition for the Condition object used to check and activate quest parts.

This file defines interfaces and concrete classes for checking quest part conditions and activating quest part transitions in the quest system. It provides flexible mechanisms for combining and customizing quest logic using various condition checks (e.g., all/any tasks completed) and activators (e.g., activate next part or multiple parts).

Dependencies:

  • <unordered_map>: Used for mapping task names to task objects.
  • <vector>: Used for storing lists of task or part names.
  • <string>: Used for task and part identifiers.
  • <memory>: Used for smart pointers (unique_ptr) to manage object lifetimes.
  • "task.hpp": Declares the Task class used in condition checks.
  • "import_export.hpp": Handles import/export macros for shared library support.
