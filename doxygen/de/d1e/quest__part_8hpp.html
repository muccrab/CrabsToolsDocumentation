Detailed Description

Contains definition for the QuestPart class that is used for managing a quest part in quests.

This header defines the QuestPart class, which represents a segment of a quest. Each QuestPart manages a collection of tasks and conditions, and supports flexible condition checking strategies. It is designed for extensibility and integration with a larger quest system.

Dependencies:

  • <unordered_map>: For storing tasks by name.
  • <vector>: For storing quest part conditions.
  • <string>: For quest part and task names.
  • <memory>: For smart pointers (shared_ptr, unique_ptr).
  • <any>: For generic update data.
  • "task.hpp": Task class definition.
  • "quest_part_condition.hpp": Quest part condition definitions.
  • "import_export.hpp": For symbol import/export macros (DLL/shared library support).
  • "quest_strategy.hpp": Visitor/strategy pattern for quest part processing.
