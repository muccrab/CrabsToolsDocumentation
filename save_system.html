Crabs Save System

The Crabs Save System is by far the simplest component of the entire framework. It is comprised of two classes that operate relatively independently from each other.

Core Classes

  • SaveSystem: This is the main interface for managing saves. It allows creating, loading, and deleting saves with minimal effort.
  • ISerializer: An interface used to serialize and deserialize save data. Currently, it saves files in JSON format with a .sv extension.

SaveSystem Constructor Options

The SaveSystem constructor supports two modes of initialization:

  • Local Path: Directly provide a path to store the saves locally.
  • App Path: Provide an app_name, a relative save_directory, and an appdata location. This will resolve to a system-compliant path such as:
    [User-specific application data folder]/appdata_location/appname/save_directory

For example, this resolves to AppData/Local on Windows and ~/.local/share on Linux.

This method ensures that users do not need to worry about storage permissions or platform-specific save paths.

Creating and Loading Saves

Once a save is created via the system, it can be saved or loaded using the ISerializer interface. Although the current implementation uses JSON format, future versions might include compression or encoding to prevent casual tampering with save files.
