Godot Integration – C++ Backend Wrappers

To connect the Crabs C++ backend with the Godot engine, a wrapper layer was implemented using GDExtensions—Godot’s native system for binding C++ classes to GDScript. This allows seamless integration and interaction between engine-side logic and native Crabs systems.

Each system in Crabs (e.g. Dialogue, Quest, Save, Serializer) exposes its core functionality to Godot via these wrappers. This approach keeps the performance and modularity of C++, while making systems accessible from Godot scripts.

This integration makes it possible to control and manipulate Crabs systems from GDScript while maintaining full power and flexibility of their native C++ implementations.
